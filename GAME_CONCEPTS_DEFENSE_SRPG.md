## ★ 1순위 컨셉 — PvP 대칭 디펜스 (Mirror Defense Duel)

> **핵심**: 양쪽이 **똑같은 재료**를 받고 → 동시에 타워+공격유닛 배치 → 시작하면 **서로의 방어진을 뚫는** 대칭 PvP 디펜스

---

### ★ PvP 대칭 디펜스 (Mirror Defense Duel)

#### 컨셉
두 플레이어가 **동일한 재료(타워+공격유닛)**를 받음. 제한 시간(60초) 안에 **방어진 구축 + 공격 편대 편성**을 동시에 함. 준비 완료 → 동시 시작 → 내 공격유닛이 상대 방어진을 뚫고, 상대 공격유닛이 내 방어진을 뚫음. **먼저 뚫린 쪽이 패배**.

#### 왜 1순위인가
- **공정성 100%**: 양쪽 동일 재료 → 순수 전략 차이로 승패 결정
- **이중 전략**: 같은 재료를 방어에 쓸까? 공격에 쓸까? → 자원 배분 딜레마
- **심리전**: 상대 배치를 모르는 상태에서 준비 → 예측과 읽기
- **바이럴**: "이 배치로 이겼다!" 리플레이 공유 → SNS 폭발
- **기존 컨셉 결합**: 미로빌더(D1) + 역전TD(D14) + 1분전쟁(S5) + Clash Royale PvP

#### 게임 플로우
```
1. 매칭: 상대방과 1:1 매칭 (또는 친구 초대)
2. 재료 지급: 양쪽 동일한 재료 세트 지급
   - 타워 재료: 화살탑x3, 대포x2, 슬로우x2, 벽x5
   - 공격 유닛: 보병x10, 궁수x5, 기마x3, 탱크x1
3. 배치 페이즈 (60초):
   - 화면 상반: 내 방어 구역 (타워+벽 배치, 미로 설계)
   - 화면 하반: 공격 편대 편성 (유닛 순서+진형)
   - 핵심 딜레마: 벽을 많이 쓰면 방어↑ but 공격 유닛에 쓸 자원↓
4. 전투 시작 (자동, 30초):
   - 화면 분할: 왼쪽=내 공격→상대 방어 / 오른쪽=상대 공격→내 방어
   - 실시간으로 양쪽 진행 상황 비교
5. 판정:
   - 먼저 기지 돌파 → 승리
   - 시간 초과 → 남은 기지 체력 비교
   - 무승부 → 서든데스 (벽 자동 파괴 시작)
6. 결과: 트로피 증감 + 리플레이 저장 + 공유
```

#### 왜 새로운가
- **타워 디펜스는 대부분 PvE** → PvP 대칭 TD는 **극소수** (Legion TD 2 정도)
- **같은 재료로 다른 결과** = 체스처럼 순수 실력 게임
- **방어와 공격을 동시에 준비** → 기존 TD에 없는 차원의 전략
- 60초 제한 → 분석 마비 방지 + 아드레날린
- 웹 브라우저 PvP 대칭 TD는 **전무** → 완전한 블루 오션

#### 핵심 중독 요소
- **자원 배분 딜레마**: "타워 1개 줄이고 기마 1기 추가하면?" → 매판 고민
- **상대 읽기**: "이 사람은 방어형일까 공격형일까?" → 심리전
- **리플레이 관전**: 내 공격이 상대 미로를 뚫는 순간 → 쾌감 폭발
- **트로피 시스템**: 랭킹 → "한 판만 더!" 무한 루프
- **메타 변화**: 시즌마다 재료 세트 변경 → 항상 새로운 전략
- **친구 대전**: "내 방어 뚫어봐!" → 우정 파괴 (=바이럴)

#### 게임 모드 확장
```
1v1 랭크전     — 트로피 기반 매칭, 시즌제
친구 대전       — 초대 코드로 즉시 대전
2v2 팀전       — 한 명은 방어 담당, 한 명은 공격 담당
솔로 캠페인    — AI 방어진 뚫기 (연습 모드)
주간 토너먼트  — 고정 재료로 최고 전략 겨루기
리플레이 극장  — 인기 대전 리플레이 관전
```

#### 수익화
- **배틀 패스** ($4.99/시즌): 시즌 한정 타워 스킨, 이모티콘, 프레임
- **리워드 광고**: 배치 시간 +15초 연장 / 패배 시 재매칭 무료
- **코스메틱 IAP**: 타워 스킨, 유닛 스킨, 승리 이펙트, 필드 테마
- **시즌 이벤트**: 특별 재료 세트 (핼러윈: 좀비 유닛, 크리스마스: 눈사람 타워)
- 예상 ARPDAU: $0.15-0.25 (PvP 게임 상위 수준)

#### 레퍼런스
- **Clash Royale**: PvP 실시간 대전 + 카드 전략 (2억 MAU)
- **Legion TD 2**: 대칭 TD PvP (Steam)
- **Bloons TD Battles**: TD 공방 교대 PvP
- **체스**: 동일 재료로 순수 전략 대결의 원조
- 웹 PvP 대칭 TD 경쟁작: **전무** → 블루 오션

#### 기술 스택
```
프론트엔드: Phaser 3 + TypeScript
멀티플레이어: Colyseus (실시간 동기화)
매칭: ELO 기반 트로피 매칭
경로탐색: PathFinding.js (미로 검증)
리플레이: 프레임별 액션 시퀀스 저장 (JSON)
에셋: Kenney Tower Defense Kit (CC0)
배포: Poki / CrazyGames (즉시 플레이)
```

#### 개발 난이도: ★★★★☆ (실시간 멀티 + 대칭 밸런싱)
#### 예상 개발 기간: 4-5주
#### 웹 경쟁: **전무 (블루 오션)**

### D1. 미로 빌더 디펜스 (Maze Builder TD)

#### 컨셉
타워를 **자유 배치**하면 적이 자동으로 경로를 재탐색. 타워가 공격도 하면서 동시에 **벽** 역할. 미로를 얼마나 길게 만드느냐가 핵심 전략.

#### 게임 플로우
```
1. 웨이브 시작 전: 골드로 타워 배치 (적 경로가 실시간 미리보기)
2. 웨이브 시작: 적이 미로를 따라 이동하며 피격
3. 웨이브 클리어: 골드 획득 → 미로 확장/업그레이드
4. 보스 웨이브: 미로를 부수는 보스 → 긴급 재건축
```

#### 왜 새로운가
- 기존 TD는 **고정 경로** → 이 게임은 **플레이어가 경로를 창조**
- 매번 다른 미로 = 무한 리플레이
- 웹 브라우저에 깔끔한 미로빌더 TD가 **아직 없음** (시장 공백)

#### 핵심 중독 요소
- "이렇게 배치하면 경로가 더 길어지지 않을까?" → 최적화 중독
- 적 경로 미리보기 = 즉각적 피드백
- 미로 스크린샷 공유 → SNS 바이럴

#### 수익화
- 리워드 광고: 추가 골드 / 특수 타워 해금
- 프리미엄 타워 스킨 IAP
- 시즌 챌린지 (주간 맵)

#### 레퍼런스
- AMazing TD, Tower Maze Defense (Steam)
- 웹 경쟁작: **없음** → 블루 오션
- 개발 난이도: ★★★☆☆ (A* 경로탐색 구현 필요)

#### 기술 스택
```
Phaser 3 + TypeScript
경로탐색: PathFinding.js (A* + JPS)
에셋: Kenney Tower Defense Kit (CC0)
```

### D2. 원타워 아이들 디펜스 (One Tower Idle)

#### 컨셉
단 **하나의 타워**만 존재. 업그레이드만으로 무한 웨이브 생존. 자동 사격, 자동 수집. 플레이어는 **업그레이드 선택**에만 집중.

#### 게임 플로우
```
1. 타워가 자동으로 적을 사격
2. 적 처치 → 골드 자동 수집
3. 골드로 업그레이드 (공격력/범위/속도/특수능력)
4. 10웨이브마다 보스 → 클리어 시 "프레스티지" 선택지
5. 오프라인에서도 골드 축적
```

#### 왜 새로운가
- 기존 TD의 **"어디에 뭘 놓을까" 고민을 제거** → 순수 업그레이드 결정만
- "The Tower" (모바일)가 이 공식으로 대성공했지만 **웹 버전이 없음**
- 아이들 게임 DAU/MAU 비율 18% (하이퍼캐주얼 10.5%보다 압도적)

#### 핵심 중독 요소
- 오프라인 진행 → 매일 열어서 축적된 골드 확인
- "다음 업그레이드까지 딱 1웨이브만 더" 루프
- 프레스티지 시스템 → 리셋해도 더 강해지는 무한 루프

#### 수익화
- 리워드 광고: 골드 2배 / 오프라인 수익 부스트
- VIP 구독: 자동 수집 범위 확대 ($2.99/월)
- 특수 무기 IAP

#### 레퍼런스
- The Tower (모바일): 아이들 TD 장르 개척
- 아이들 게임 시장: $25억 (2024), CAGR 10%
- 개발 난이도: ★★☆☆☆ (단일 타워 + 업그레이드 트리)

#### 기술 스택
```
Phaser 3 + TypeScript (또는 순수 React+Canvas)
BigNumber.js (큰 숫자 처리)
localForage (오프라인 세이브)
```

### D3. 리듬 디펜스 (Beat Defense)

#### 컨셉
타워가 **음악 비트에 맞춰** 사격. 플레이어가 비트에 맞춰 탭하면 **크리티컬 데미지**. 음악 장르가 바뀌면 게임 스타일도 변화.

#### 게임 플로우
```
1. BGM 시작 → 적이 비트에 맞춰 행진
2. 타워 자동 사격 (기본 데미지)
3. 플레이어가 비트에 맞춰 탭 → 크리티컬 (2-3배)
4. 콤보 유지 → 특수 스킬 게이지 충전
5. 스테이지 = 한 곡 (3-4분)
```

#### 왜 새로운가
- 리듬 게임 + TD = **거의 미개척 조합** (Bass Defense가 2025년 첫 시도)
- 음악이 게임 자체 → 별도 BGM 불필요 (음악 = 메커닉)
- 비트 맞추기 실패해도 타워가 기본 사격 → **실패 부담 0**

#### 핵심 중독 요소
- 리듬 게임 특유의 "풀 콤보" 쾌감
- 음악 자체가 중독 → 게임 종료 후에도 곡이 머릿속에
- "이 곡은 S랭크 받아야지" → 재도전

#### 수익화
- 추가 음악팩 IAP ($1.99/팩)
- 리워드 광고: 특수 타워 해금
- 시즌 뮤직 페스티벌 이벤트

#### 레퍼런스
- Bass Defense (Steam 2025): 장르 첫 시도
- 웹 경쟁작: **전무** → 완전 블루 오션
- 개발 난이도: ★★★☆☆ (비트 동기화 시스템)

#### 기술 스택
```
Phaser 3 + Web Audio API
비트 감지: Tone.js 또는 커스텀 BPM 동기화
음악: OpenGameArt 칩튠 or Suno AI 생성
```

### D4. 요리 디펜스 (Kitchen Defense)

#### 컨셉
배고픈 몬스터가 식당으로 몰려옴. **주방 기구가 타워** (프라이팬=근거리, 오븐=범위, 냉장고=슬로우). 요리를 완성해서 몬스터에게 던지면 격퇴. 미로 배치도 가능.

#### 게임 플로우
```
1. 주방 배치: 조리대, 오븐, 프라이팬 등을 그리드에 배치
2. 재료 자동 수집 → 레시피 자동 조리
3. 완성된 음식이 몬스터에게 자동 투척
4. 고급 레시피 = 강력한 공격 (라면 < 스테이크 < 한정식)
5. 미로처럼 배치해서 이동 경로 늘리기
```

#### 왜 새로운가
- **요리 + TD** 조합은 Kitchen Crisis(Steam)만 존재, 캐주얼 웹 버전 **없음**
- 요리 게임의 친숙함 → 비게이머도 "이건 알겠다"
- 한국 음식 테마: 떡볶이 타워, 김치찌개 범위공격, 삼겹살 화염 데미지

#### 핵심 중독 요소
- 레시피 해금 수집욕
- 음식 조합 실험 → "이걸 합치면 뭐가 되지?"
- 귀여운 비주얼 + 한국 음식 → 공유 욕구

#### 수익화
- 프리미엄 레시피 IAP
- 리워드 광고: 레어 재료 획득
- 시즌 이벤트 (크리스마스 디저트 방어전)

#### 레퍼런스
- Kitchen Crisis (Steam): 컨셉 증명
- 요리 게임 시장 CAGR 6.6% (2035까지)
- 개발 난이도: ★★★☆☆ (레시피 시스템 + TD 로직)

#### 기술 스택
```
Phaser 3 + TypeScript
경로탐색: PathFinding.js
에셋: 음식 픽셀아트 (Pixelorama 제작 or AI)
```

### D5. 로그라이크 머지 디펜스 (Roguelike Merge TD)

#### 컨셉
웨이브 사이에 랜덤 타워 3개 중 1개 선택. 같은 타워 2개를 **머지**하면 상위 타워로 진화. 매 런마다 다른 빌드. 죽으면 영구 업그레이드 후 재시작.

#### 게임 플로우
```
1. 웨이브 시작 → 적 침입
2. 웨이브 클리어 → 타워 3개 중 1개 선택 (로그라이크 보상)
3. 같은 타워 2개 드래그 → 머지 (상위 등급)
4. 시너지 조합: 불+불=화염폭풍, 불+얼음=증기타워
5. 사망 → 메타 업그레이드 (시작 골드+, 선택지 4개로 증가)
```

#### 왜 새로운가
- **머지(2048) + 로그라이크(뱀서) + TD** 3장르 교차
- Random Dice가 머지+TD를 증명했지만, **로그라이크 런 구조** 추가는 신선
- 매 런이 다름 → 무한 리플레이 가치

#### 핵심 중독 요소

### D6. 그림 디펜스 (Draw Defense)

#### 컨셉

#### 게임 플로우
```
1. 잉크 게이지 충전 상태에서 시작
2. 적 진입 방향 표시 → 선을 그어 벽/공격 라인 배치
3. 적이 벽에 부딪히면 벽 내구도 소모 + 데미지
4. 직선=관통 공격, 원=범위 방어, 지그재그=슬로우
5. 잉크 소진 → 적 처치로 잉크 회복
6. 보스: 벽을 한방에 부숨 → 실시간 재드로우
```

- "이런 모양으로 그리면 더 효율적이지 않을까?" → 최적화 중독
- 그림 공유 → SNS 바이럴 ("내 방어벽 봐봐")

### D7. 테트리스 디펜스 (Tetris TD)

#### 컨셉
**테트리스 블록**이 하늘에서 떨어짐. 블록을 배치하면 **지형+타워**가 동시에 생성. 한 줄이 완성되면 그 줄 전체가 공격 발동. 블록 배치 자체가 미로이자 무기.

#### 게임 플로우
```
1. 테트리스처럼 블록이 위에서 하강
2. 좌우 이동/회전으로 배치

4. 한 줄 완성 → 전체 줄이 폭발 공격 (테트리스 라인 클리어)
5. 적은 좌→우로 블록 사이를 통과하려 함
6. 블록이 천장까지 쌓이면 게임 오버
```

#### 왜 새로운가
- **테트리스 + TD** 조합은 Emberward가 타일 개념만 차용 → 순수 테트리스 룰 적용은 **미개척**
- 테트리스 조작법은 전 세계가 이미 앎 → 학습 비용 0
- 라인 클리어 = 범위 공격이라는 직관적 연결

#### 핵심 중독 요소
- 테트리스 특유의 "한 줄만 더!" 중독
- 블록 배치가 전략(미로) + 퍼즐(테트리스) 이중 만족
- T-스핀으로 라인 클리어 = 특수 공격 → 고수만의 쾌감

#### 수익화
- 블록 스킨 IAP (네온, 레트로, 우주)
- 리워드 광고: 다음 블록 미리보기 / 홀드 기능

#### 레퍼런스
- 테트리스: 5억+ 유저, 역대 최다 판매 게임
- Emberward: 타일 기반 TD 컨셉
- 개발 난이도: ★★★☆☆ (테트리스 엔진 + TD 로직 결합)

### D8. 날씨 디펜스 (Weather Defense)

#### 게임 플로우
```
1. 4가지 날씨 카드: ☀️태양 🌧️비 ❄️눈 🌪️바람

3. 적이 해당 구역 진입 시 날씨 효과 적용
4. 조합: 🌧️+❄️=빙판(슬립), ☀️+🌧️=무지개(아군버프), 🌪️+❄️=블리자드
5. 마나로 날씨 유지 — 마나 소진되면 날씨 소멸
6. 웨이브 간 날씨 카드 업그레이드
```

#### 왜 새로운가

- 날씨 조합 = 화학 반응 느낌 → 발견의 재미
- 비주얼이 압도적 (비, 눈, 번개 이펙트)

#### 핵심 중독 요소

- 날씨 이펙트 자체가 시각적 쾌감
- 영역 관리 → 체스처럼 공간을 읽는 전략

#### 수익화

- 리워드 광고: 마나 충전

### D9. 진화하는 적 디펜스 (Adaptive Defense)

#### 컨셉
적이 **플레이어 전략에 맞춰 진화**. 화염 타워를 많이 쓰면 내화 적 등장, 슬로우 타워엔 면역 적 진화. 한 가지 전략에 안주하면 망함 → 매 웨이브 전략 변경 강제.

#### 게임 플로우
```
1. 기본 타워 배치 → 첫 웨이브 클리어
2. 적 AI 분석: "화염 타워 60% 사용 감지"
3. 다음 웨이브: 내화(Fire-Resistant) 적 30% 혼합
4. 플레이어 대응: 얼음 타워 추가
5. 적 AI: "얼음 타워 증가 감지" → 내한 적 등장
6. 적 진화 로그 표시 → 플레이어가 대응 전략 수립
```

#### 왜 새로운가
- 기존 TD는 정해진 웨이브 패턴 → 이 게임은 **적이 배운다**
- 메타가 고정되지 않음 → 최적해가 없는 TD
- "내가 바꾸면 적도 바뀐다" → 끝없는 두뇌 싸움

#### 핵심 중독 요소
- "적이 내 전략을 읽었다!" → 긴장감
- 진화 로그 = 적의 학습 과정을 관찰하는 재미
- 밸런스 깨기 불가 → 항상 도전적

#### 수익화
- 리워드 광고: 적 진화 로그 미리보기
- 프리미엄 캠페인 (특수 진화 패턴)
- 스킨 IAP

#### 레퍼런스

- Nomad Tower X: 적 적응 시스템 실험
- 개발 난이도: ★★★★☆ (적응형 AI 로직)

#### 기술 스택
```
Phaser 3 + TypeScript
적 AI: 플레이어 행동 추적 + 가중치 기반 유닛 생성
```

### D13. 호위 디펜스 (Escort Defense)

### D15. 주사위 타워 디펜스 (Dice TD)

#### 컨셉
타워 대신 **주사위를 굴려서** 배치. 주사위 눈금 = 타워 등급+종류. 같은 눈금 2개 합치면 상위 눈금으로 머지. 운 + 전략의 완벽한 균형.

#### 게임 플로우
```
1. 웨이브 시작 전: 주사위 3개 굴림
2. 나온 눈금의 타워가 생성 (1=약한 화살탑, 6=강한 마법탑)
3. 같은 눈금 2개 머지 → 상위 눈금 (1+1=2, 3+3=4)
4. 6+6 = ★특수 타워 (랜덤 레전더리)
5. 굴림 횟수 제한 → 전략적 머지 필수
6. 보스 웨이브: 추가 굴림 찬스
```

#### 왜 새로운가
- **Random Dice** 공식을 캐주얼 TD로 재해석
- 주사위 굴림 = 시각적 재미 + 가챠 도파민
- 운이 나쁘면 전략으로 커버 → 스킬과 운의 밸런스

#### 핵심 중독 요소

- "6이 나와라!" → 도박 심리
- 머지 시스템 → 2048 중독
- 레전더리 타워 등장 시 화면 이펙트 폭발

#### 수익화
- 프리미엄 주사위 스킨 (골드, 크리스탈, 네온)
- 리워드 광고: 추가 굴림
- 시즌 한정 레전더리 타워

#### 레퍼런스
- Random Dice: 1억+ 다운로드
- Dice Tower Defense (itch.io): 개념 실험
- 개발 난이도: ★★☆☆☆ (주사위 물리 + 머지 + TD)

#### 기술 스택
```
Phaser 3 + TypeScript
주사위: 3D CSS 또는 스프라이트 애니메이션

```

```
5. "전투!" → 30초 자동전투 (병사 수 = 시각적 체력)
6. 영웅 사망 → 휘하 병사 전원 패닉 도주
```

```
1. 유닛에 주사위 3개 장착 (이동/공격/방어)
2. 턴 시작: 주사위 3개 동시 굴림
3. 이동 주사위 4 → 4칸 이동 가능
4. 공격 주사위 5 vs 적 방어 주사위 3 → 2 데미지

6. 주사위 커스텀: 대장간에서 면 교체 (1면을 6으로!)
```

#### 왜 새로운가
- SRPG의 확률을 **눈에 보이는 주사위**로 물리화
- FE의 명중률 80%보다 **주사위 굴리는 게 100배 재밌음**
- 주사위 커스텀 = 장비 시스템의 완전한 대체
- Dicey Dungeons가 주사위RPG를 증명 → 전술 전투 확장은 새로움

#### 핵심 중독 요소

- 트리플 나오는 순간 → 로또 쾌감
- 주사위 면 커스텀 → 빌드 실험 무한
- "운이 나빠도 전략으로 커버" → 스킬 보상 있음

#### 수익화
- 레어 주사위 세트 IAP (크리스탈, 골드, 다이아)

### S14. 동맹과 배신 (Alliance & Betrayal)

#### 컨셉
**4인 멀티플레이어** 전쟁. 매 라운드 **동맹/배신** 투표 → 동맹하면 같이 싸우고, 배신하면 뒤통수. 최후의 1인만 승리. 외교가 전투보다 중요.

#### 게임 플로우
```
1. 4명 각자 영토 + 군대 보유
2. 라운드 시작: 비밀 투표 (동맹할 상대 1명 선택)
3. 서로 동맹 선택 → 군사력 합산, 공동 공격
4. 한쪽만 동맹, 상대는 배신 → 배신한 쪽이 기습 보너스
5. 양쪽 다 배신 → 양쪽 다 데미지
6. 3명 탈락 → 최후 1인 승리
```

#### 왜 새로운가
- **죄수의 딜레마**를 게임으로 → 심리전이 핵심
- Among Us의 사회적 재미 + 전쟁 시뮬의 전략
- 친구와 함께 하면 **우정 파괴 게임** → 바이럴
- 랑그릿사의 진영 분기를 **실시간 멀티로 재해석**

#### 핵심 중독 요소
- "이번에는 진짜 동맹할게" → 거짓말의 쾌감
- 배신 성공 시 역전극 → 드라마
- 채팅으로 외교 → 마피아 게임급 몰입
- 매 판 다른 인간관계 = 무한 리플레이

#### 수익화
- 이모티콘/캐릭터 스킨 IAP
- 방 꾸미기 (프리미엄 맵)
- 배틀 패스

#### 레퍼런스
- Among Us: 사회적 추리의 교과서
- Risk: 동맹과 배신의 보드게임 원조
- 개발 난이도: ★★★☆☆ (멀티 투표 시스템 + 간단한 전투)

#### 기술 스택
```
Phaser 3 + Colyseus (실시간 멀티) + TypeScript
투표: 비밀 투표 서버 로직
채팅: Socket.io 또는 Colyseus 메시지
```

### S15. 한 줄 전쟁 (One Line War)

#### 컨셉
전장이 **가로 한 줄**. 좌→우로 아군, 우→좌로 적군이 행진. 유닛 배치 순서와 간격이 전략의 전부. 극도로 미니멀하지만 깊은 상성 전략.

#### 게임 플로우
```
1. 화면 = 가로 1줄 전장
2. 왼쪽에서 아군 유닛 배치 (순서 결정)
3. 오른쪽에서 적 유닛 행진
4. 만나면 자동 전투 (상성: 🗡️>🏹>🛡️>🗡️)
5. 아군이 우측 끝 도달 → 승리
6. 핵심: 탱크를 앞에? 딜러를 앞에? 간격은?
```

#### 왜 새로운가
- **1차원 SRPG** → 극한의 단순화
- 2D 전장의 복잡함을 **순서 결정 하나로** 압축
- Epic War 시리즈의 공식을 **상성+간격 전략으로 진화**
- 한 손 스크롤로 플레이 가능 → 모바일 완벽 적합

#### 핵심 중독 요소
- 유닛 순서 1개 바꿨더니 결과가 완전히 달라짐
- 한 판 10초 → "한 번 더!" 무한 반복
- 적 패턴 읽기 → "아 이번엔 탱크 먼저!"
- PvP: 상대 배치를 예측하는 심리전

#### 수익화
- 유닛 스킨 IAP
- 리워드 광고: 유닛 추가 1개
- PvP 랭킹 시즌

#### 레퍼런스
- Epic War: 1줄 전쟁의 원조
- Stick War: 스틱맨 1줄 전투
- 개발 난이도: ★☆☆☆☆ (가장 간단한 구조)

#### 기술 스택
```
Phaser 3 또는 순수 React + TypeScript
전투 로직: 1D 배열 충돌 판정
에셋: 이모지 또는 미니멀 도트
```
