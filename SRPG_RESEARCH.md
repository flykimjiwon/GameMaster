### 1. Fire Emblem Heroes (파이어 엠블렘 히어로즈)
**서브장르**: 클래식 SRPG + 모바일 가챠

| 항목 | 내용 |
|------|------|
| **플랫폼** | iOS, Android |
| **개발사** | Intelligent Systems / Nintendo (~200명 규모) |
| **핵심 메카닉** | 8x6 소규모 그리드 전투, 무기 상성 삼각관계, 캐릭터 소환(가챠), 아레나 PvP, 정기적 이벤트 |
| **수익 모델** | F2P + 가챠 (오브 구매). 한정 배너, 듀얼 피크업 등으로 지출 유도 |
| **매출/인기도** | 누적 매출 $1.3B 이상. 닌텐도 모바일 최고 매출작. 2024년 연매출 ~$48M (피크 대비 하락세) |
| **아트 스타일** | 2D 일러스트 (다수 유명 일러스트레이터 참여) + SD 스프라이트 전투 |
| **참고할 포인트** | **소규모 맵(8x6)으로도 충분히 깊은 전략 가능.** 모바일에서 SRPG를 성공시킨 핵심은 한 판 3~5분의 짧은 플레이 타임. 웹 SRPG에도 이 접근이 유효. 캐릭터 수집 욕구를 자극하는 일러스트 퀄리티가 매출 핵심 동력. |

### 3. Into the Breach (인투 더 브리치)
**서브장르**: 미니멀 SRPG + 로그라이크 ★ 인디 대표 성공작

| 항목 | 내용 |
|------|------|
| **플랫폼** | PC (Steam), Switch, iOS, Android (Netflix) |
| **개발사** | Subset Games (**단 2명** - Justin Ma, Matthew Davis) |
| **핵심 메카닉** | 8x8 그리드, 3유닛 조합, **적의 다음 행동이 미리 보임** (완전 정보 게임), 턴마다 퍼즐처럼 최적해를 찾는 구조, 로그라이크 런 반복 |
| **수익 모델** | 프리미엄 $15 (Netflix 무료 포함) |
| **매출/인기도** | Steam 100~200만 소유자. PC Gamer 93점, GameSpot 90점. FTL 수익으로 6년간 자가 펀딩 가능했을 정도의 상업적 성공 |
| **아트 스타일** | **미니멀 픽셀아트** - 깔끔하고 명확한 UI, 적은 에셋으로 최대 가독성 |
| **참고할 포인트** | **인디 SRPG의 교과서.** 2명이 만든 게임이 수백만 장 판매. 핵심 교훈: (1) 유닛 수를 3개로 극단적으로 줄여 의사결정 밀도를 높임, (2) "적 행동 미리보기"라는 하나의 혁신으로 장르를 재정의, (3) 미니멀 아트로 개발 비용 최소화, (4) 로그라이크 구조로 리플레이 가치 극대화. **Phaser로 구현 완전 가능한 스케일.** |

| **아트 스타일** | **2D 픽셀아트** (16~32bit 스타일) + 다크 판타지 분위기 |

| 항목 | 내용 |

| **개발사** | Suspicious Developments (**3명**) |
| **핵심 메카닉** | 소규모 퍼즐형 전투, **"되감기(Undo)" 시스템** - 행동을 자유롭게 되돌리며 최적 해법 탐색, 적을 창문 밖으로 밀어내기 등 환경 인터랙션, 스토리 중심 |
| **수익 모델** | 프리미엄 $20 |
| **매출/인기도** | **출시 3주 만에 10만장+ 판매.** Steam "압도적 긍정" (98%). IGF Excellence in Design 수상. Steam 8월 프리미엄 매출 TOP 10 진입 |
| **아트 스타일** | 2D 일러스트 + 깔끔한 UI (카툰 스타일) |

## 2024~2026 SRPG 트렌드 요약

### 1. 하이브리드 장르의 시대
순수 SRPG보다 **SRPG + X** 형태가 주류. 로그라이크, 덱빌딩, 타워디펜스, 오토배틀 등과의 결합이 시장을 지배.

### 2. 소규모 팀의 황금기
1~3명 팀이 50만~200만 판매를 달성하는 사례 다수. **Into the Breach(2명), Symphony of War(1~3명), Tactical Breach Wizards(3명), Fell Seal(2명)**. 핵심은 "하나의 혁신적 메카닉 + 견고한 기본기".

### 3. 미니멀 디자인의 부상
대규모 맵/대군 전투보다 **소규모 맵 + 소수 유닛 + 높은 의사결정 밀도**가 트렌드. Into the Breach(3유닛, 8x8), TBW(소규모 퍼즐맵)가 대표.

### 4. 로그라이크 통합이 표준화
일회성 캠페인보다 **반복 플레이 가능한 로그라이크/로그라이트 구조**가 리텐션과 콘텐츠 효율 모두에서 우위. The Last Spell, Into the Breach, StarVaders 등.

### 5. 웹/Phaser SRPG의 가능성
- OpenAI가 Phaser로 택티컬 RPG 프로토타입을 브라우저에서 구현한 사례(2026.3)
- Defender's Quest가 웹(Flash)에서 시작해 상업 게임으로 성장
- 2D 픽셀아트 + 그리드 기반 전투는 Phaser에 최적화된 포맷
- **웹 SRPG의 킬러 조합**: 소규모 맵 + 로그라이크 + 카드/덱빌딩 메카닉

### 6. 아트 디렉션이 곧 차별화
Reverse: 1999(아르데코), Arknights(다크 SF 애니메이션) 등 **독특한 아트 디렉션**이 포화 시장에서의 핵심 차별점. 인디는 "예쁜 것"보다 "개성 있는 것"이 중요.

## 인디 개발자를 위한 핵심 시사점

### Phaser/웹 SRPG 개발 시 추천 방향

```
[최소 리스크 / 최대 효과 조합]

1. 그리드 크기: 6x6 ~ 8x8 (FE Heroes의 8x6 참고)
2. 유닛 수: 3~6개 (Into the Breach의 3유닛 철학)
3. 핵심 혁신 1개: 되감기/적 미리보기/카드 이동 등
4. 로그라이크 구조: 런 기반 반복 플레이
5. 아트: 미니멀 픽셀 or 개성 있는 2D 일러스트
6. 수익: 프리미엄 $10~20 or F2P + 코스메틱/배틀패스
7. 플랫폼: 웹(PC 브라우저) → 모바일(PWA/하이브리드) → Steam
```

### 피해야 할 함정
- 대규모 맵 + 대군 전투 (개발 비용 폭발)
- 풀 3D (인디에 비현실적)
- 가챠 메카닉 (대형 IP 없이는 유저 확보 어려움)
- 순수 캠페인 only (리플레이 부족 → 짧은 수명)
