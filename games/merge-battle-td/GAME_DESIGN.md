# Merge Battle TD — 상세 기획서

> 머지 PvP 타워디펜스 대전 | 프로토타입 v0.1 | 1턴 1판짜리 MVP

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 머지(합성) + PvP 타워디펜스 대전 |
| **플랫폼** | 웹 (브라우저) → 텔레그램, 디스코드, Poki |
| **타겟** | 캐주얼~미드코어, PvP 좋아하는 유저 |
| **세션 시간** | 1판 3~5분 |
| **조작** | 드래그 앤 드롭 (터치/마우스) |
| **엔진** | Phaser 3 + TypeScript |
| **백엔드** | Next.js + Supabase Realtime (PvP 동기화) |
| **핵심 컨셉** | 같은 재료 → 디펜스 구축 → 상대 디펜스 공격 (지원님 오리지널) |

## 2. 핵심 게임 루프 (2페이즈 턴제)

```
[매칭] → 양쪽 플레이어에게 동일한 타워 조각 배분
    ↓
━━━ Phase 1: BUILD (30초) ━━━
│ 받은 타워 조각을 그리드에 배치
│ 같은 종류 2개를 합성(머지) → 상위 타워로 진화
│ 타워 위치 = 경로에 대한 방어 전략
    ↓
━━━ Phase 2: ATTACK (자동, 30초) ━━━
│ 내가 만든 공격 유닛이 상대 디펜스로 진격
│ 상대가 만든 공격 유닛이 내 디펜스로 진격
│ 양쪽 동시 진행 (스플릿 스크린)
    ↓
[라운드 결과] → 관통 유닛 수 = 데미지
    ↓
━━━ 다음 턴 ━━━
│ 새 타워 조각 배분 + 코인으로 추가 구매
│ 기존 타워 유지 + 새 타워 추가/머지
    ↓
[반복... HP 0이 되면 패배]
```


## 5. 타워 시스템

### 5A. 머지 규칙

```
Tier 1 + Tier 1 = Tier 2
Tier 2 + Tier 2 = Tier 3
(프로토타입은 Tier 3까지)
```

드래그해서 같은 Tier 타워 위에 놓으면 머지. 다른 Tier면 위치 교환.

### 5B. 프로토타입 타워 3종

| 이름 | 색상 | Tier 1 | Tier 2 | Tier 3 |
|------|------|--------|--------|--------|
| **아처** | 🟢 초록 | DMG 5, 사거리 3칸, 공속 1.0s | DMG 12, 사거리 3, 공속 0.8s | DMG 30, 사거리 4, 공속 0.5s |
| **캐논** | 🔴 빨강 | DMG 15, 사거리 2, 공속 2.0s, 범위공격 | DMG 35, 사거리 2, 공속 1.5s | DMG 80, 사거리 3, 공속 1.0s |
| **슬로우** | 🔵 파랑 | DMG 2, 사거리 2, 감속 30% | DMG 5, 사거리 3, 감속 50% | DMG 10, 사거리 3, 감속 70% |

### 5C. 타워 조각 배분

프로토타입:
- 빌드 페이즈 시작 시 **랜덤 Tier 1 타워 5개** 지급
- 종류는 3종 중 랜덤 (편중 가능 → 머지 전략)

## 8. PvP 구조 (추후 — 설계만)

### 8A. 매칭 플로우

```
[매칭 큐] → Supabase Realtime Channel
    ↓
[매칭 성공] → 양쪽에 동일한 seed 전송
    ↓
[Phase 1: BUILD] → 각자 독립적으로 빌드 (30초)
    ↓
[Phase 1 끝] → 양쪽 배치 데이터를 서버로 전송
    ↓
[Phase 2: ATTACK] → 서버에서 시뮬레이션 or 클라이언트 동기화
    ↓
[결과] → 양쪽 통과 적 수 비교 → 적게 통과시킨 쪽이 승리
```

### 8B. 동기화 전략

| 방식 | 장점 | 단점 | 선택 |
|------|------|------|------|
| **락스텝** | 완벽 동기화 | 구현 복잡 | X |
| **클라 시뮬 + 결과 검증** | 간단, 체감 좋음 | 치팅 가능 | **✓ 프로토** |
| **서버 시뮬** | 치팅 방지 | 서버 부하 | 추후 |

프로토타입: 빌드 데이터만 교환 → 각 클라이언트에서 시뮬레이션 → 결과 비교

## 9. 기술 구조

### 9A. 프로젝트 구조

```
games/merge-battle-td/
├── package.json
├── vite.config.ts
├── tsconfig.json
├── index.html
├── public/
│   └── assets/
│       ├── sprites/        # 타워, 적 스프라이트
│       └── audio/          # 효과음
└── src/
    ├── main.ts             # Phaser 게임 초기화
    ├── config.ts           # 게임 설정 상수
    ├── scenes/
    │   ├── BootScene.ts    # 에셋 로딩
    │   ├── BuildScene.ts   # Phase 1: 타워 배치
    │   ├── BattleScene.ts  # Phase 2: 전투 시뮬
    │   ├── UIScene.ts      # HUD 오버레이
    │   └── ResultScene.ts  # 결과 화면
    ├── entities/
    │   ├── Tower.ts        # 타워 기본 클래스
    │   ├── TowerFactory.ts # 타워 생성/머지 팩토리
    │   ├── Enemy.ts        # 적 기본 클래스
    │   └── Projectile.ts   # 투사체
    ├── systems/
    │   ├── GridSystem.ts     # 5x3 그리드 관리
    │   ├── MergeSystem.ts    # 머지(합성) 로직
    │   ├── PathSystem.ts     # 적 이동 경로
    │   ├── WaveSystem.ts     # 웨이브 스폰
    │   ├── CombatSystem.ts   # 타워 공격 로직
    │   └── DragDropSystem.ts # 드래그 앤 드롭
    └── utils/
        ├── helpers.ts
        └── constants.ts
```

### 9B. 핵심 기술 포인트

| 항목 | 구현 방식 |
|------|----------|
| **그리드** | 2D 배열, 셀 좌표 ↔ 월드 좌표 변환 |
| **드래그 앤 드롭** | Phaser Input.setDraggable() |
| **머지 판정** | 드롭 시 대상 셀의 타워 Tier/Type 비교 |
| **경로** | 웨이포인트 배열, Phaser.Math.Interpolation |
| **타워 타겟팅** | 사거리 내 가장 앞선 적 (First 방식) |
| **투사체** | Phaser Arcade Physics, moveToObject |
| **PvP (추후)** | Supabase Realtime Channels |

## 11. 머지 UX 상세

### 11A. 드래그 앤 드롭 플로우

```
1. 하단 타워 조각 터치/클릭 → 드래그 시작
2. 그리드 셀 위로 이동 → 셀 하이라이트
   - 빈 셀: 초록 하이라이트 (배치 가능)
   - 같은 Tier 타워 셀: 노란 하이라이트 (머지 가능)
   - 다른 Tier 타워 셀: 파란 하이라이트 (교환)
   - 경로 셀: 빨간 하이라이트 (배치 불가)
3. 드롭
   - 빈 셀 → 타워 배치
   - 같은 Tier → 머지 애니메이션 → 상위 Tier 등장
   - 다른 Tier → 위치 교환
   - 경로 → 원위치 복귀
```

### 11B. 머지 이펙트

```
[Tier1 🟢] + [Tier1 🟢] → 💫 파티클 → [Tier2 🟢🟢] (크기 1.5배)
[Tier2 🟢🟢] + [Tier2 🟢🟢] → ⭐ 파티클 → [Tier3 🟢🟢🟢] (크기 2배, 빛남)
```
